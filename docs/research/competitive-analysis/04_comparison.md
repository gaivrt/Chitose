## Chitose 领先维度

- **架构质量**：LiveKit Agents 框架天然支持流式处理，模块化清晰
- **响应延迟**：LLM 流式 → TTS 流式 → LiveKit 推流，端到端 <500ms
- **Lip Sync**：FFT 频率分析取低频 30% 人声段，优于二值和纯 RMS
- **跨平台**：无 VTube Studio 依赖，浏览器即可运行
- **依赖管理**：精简依赖，安装部署简单
- **模型资源**：芊芊模型自带 31 个表情预设（星星眼、脸红、生气、比心等）、
  身体旋转 XYZ、呼吸、道具切换（扇子、话筒、笔记本等），
  以及丰富的面部细节参数（眼珠、眉毛、嘴巴多维度控制），
  但目前仅接入了嘴巴开合，大量能力未被利用

## Chitose 落后维度

- **功能丰富度**：仅支持单一 LLM/TTS/STT provider
- **直播平台**：无任何直播平台集成
- **记忆系统**：无持久化记忆
- **中文 STT**：Deepgram nova-3 中文识别效果不佳

## 专题：Lip Sync 技术对比

| 方案 | 项目 | 原理 | 优点 | 缺点 |
|------|------|------|------|------|
| 二值开关 | Luna AI | 说话=开，不说话=关 | 实现最简单 | 完全不自然 |
| RMS 音量 | Open-LLM-VTuber | 音频 RMS → 嘴巴开合 | 有动态变化 | 噪音也触发，无口型区分 |
| FFT 频率 | Chitose | 低频段能量 → 嘴巴开合 | 人声提取更准 | 仍无元音区分 |
| Phoneme→Viseme | 业界最佳 | 语音→音素→口型映射 | 最自然 | 实现复杂，需额外模型 |

Chitose 当前处于第三级，升级到 Phoneme→Viseme 是中期目标。

## 专题：TTS 延迟对比

| 项目 | 流程 | 瓶颈 |
|------|------|------|
| Luna AI | LLM 完整回复 → TTS 合成 → 音频队列播放 | 全阻塞，队列管理有 bug |
| Open-LLM-VTuber | LLM 完整回复 → TTS 合成 → base64 → WebSocket | 全阻塞，编码传输额外开销 |
| Chitose | LLM 流式 → TTS 流式 → LiveKit 实时推流 | 几乎无瓶颈，<500ms |

## 专题：表情与动作系统对比

| 项目 | 表情控制 | 身体动作 | 道具系统 |
|------|---------|---------|---------|
| Luna AI | VTube Studio 转发，依赖外部软件 | VTube Studio 控制 | 无 |
| Open-LLM-VTuber | 前端 Live2D 基础表情切换 | 无 | 无 |
| Chitose | 31 个 exp3 预设（未接入 AI） | 身体旋转 XYZ + 呼吸参数（未接入） | 12+ 道具切换（未接入） |

Chitose 的芊芊模型资源远超竞品使用的通用模型，关键差距在于 AI 驱动层：

- **表情**：31 个预设覆盖情绪（生气、脸红、流汗）、特效（星星眼、爱心眼）、
  道具（话筒、扇子、笔记本、打游戏），需要 LLM 情感分析来触发
- **身体动作**：ParamAngle XYZ + ParamBodyAngle XYZ 可实现头部和身体摆动，
  需要 idle 动画系统或语音节奏驱动
- **面部细节**：眼珠追踪、眉毛、嘴巴多参数（张合 + 变形 + Jaw），
  可大幅提升 lip sync 自然度
